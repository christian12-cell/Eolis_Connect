🏗️ Vue d'ensemble de l'architecture

Eolis Connect est une application client-serveur classique découpée en 3 couches :

🖥️ Frontend — Next.js (Vercel)

Ce que l'utilisateur voit et avec quoi il interagit. Tourne dans le navigateur (Chrome, Safari, etc.) sur le téléphone ou l'ordinateur. Code : TypeScript + React.

⚙️ Backend — FastAPI (Railway)

Le cerveau. Reçoit les requêtes du frontend, applique les règles métier, lit/écrit dans la base de données. Code : Python.

🗄️ Base de données — PostgreSQL (Neon)

Le disque dur. Stocke toutes les données de façon permanente : utilisateurs, tickets, messages, crédits, etc. Langage : SQL.

Chaque fois que l'appli fait quelque chose (créer un ticket, envoyer un message…), voici ce qui se passe :

  1. Le frontend appelle l'API backend via HTTP (ex: POST /api/tickets)
  2. Le backend vérifie l'identité (JWT token), valide les données, puis écrit/lit en base
  3. La base renvoie les données → le backend les formate en JSON → le frontend les affiche

⚛️ Next.js — C'est quoi exactement ?

Next.js est un framework basé sur React. React c'est la librairie de base (composants, état), Next.js ajoute le routing, l'optimisation, le build. Pense à React comme un moteur de voiture, et Next.js comme la voiture complète.

Ce qu'il faut comprendre sur notre version (Next.js 16)

⚠️
Règle critique : 'use client' en haut de CHAQUE fichier Dans Next.js moderne, les composants peuvent tourner côté serveur (pas de localStorage, pas de browser APIs). Comme on utilise localStorage pour l'auth, tous nos fichiers doivent être client-side. Sans 'use client' → crash au démarrage.

App Router — Le routing par dossiers

Dans Next.js 16, l'URL correspond au chemin des dossiers :

src/app/[locale]/ ↑ [locale] = "fr" ou "en" dans l'URL (client)/mes-demandes/page.tsx → /fr/mes-demandes (client)/mes-demandes/[id]/page.tsx → /fr/mes-demandes/abc-123 (agent)/agent/dashboard/page.tsx → /fr/agent/dashboard (admin)/admin/utilisateurs/page.tsx → /fr/admin/utilisateurs

Les dossiers entre parenthèses (client) (agent) sont des groupes de route — ils organisent le code sans apparaître dans l'URL.

params = Promise — Règle importante Next.js 16

Dans les anciennes versions, params.locale était direct. En Next.js 16, c'est une Promise. Si tu oublies le .then() → la page ne connaît jamais le locale et n'affiche rien :

// ❌ Ancienne façon (cassée en Next.js 16)
export default function Page({ params }) {
  const locale = params.locale  // undefined !
}

// ✅ Bonne façon
export default function Page({ params }: { params: Promise<{ locale: string }> }) {
  const [locale, setLocale] = useState('fr')
  useEffect(() => { params.then(p => setLocale(p.locale)) }, [params])
}

🧩 Composants React — Concepts clés

Un composant = une fonction qui retourne du HTML

// Composant simple

Règle critique : sous-composants EN DEHORS du composant parent

🔴
Bug perte de focus sur les inputs ! Si tu définis un composant (ex: function InputField()) INSIDE le composant parent, React recrée ce composant à chaque re-render → l'input perd le focus à chaque frappe. TOUJOURS définir les sous-composants AVANT le export default function. 
// ❌ MAUVAIS — LogisticsFields défini INSIDE NouvelleDemandePage
export default function NouvelleDemandePage() {
  function LogisticsFields() { ... }  // ← remonte à chaque render !
  return <LogisticsFields />
}

// ✅ BON — LogisticsFields défini AVANT
function LogisticsFields() { ... }  // ← stable, jamais recréé

export default function NouvelleDemandePage() {
  return <LogisticsFields />
}

🧠 État & données locales — useState, useEffect, useRef

useState — stocker une valeur qui change

// Déclarer un état
const [text, setText] = useState('')
//    ↑ valeur  ↑ fonction pour changer   ↑ valeur initiale

// Changer la valeur

// Changer basé sur l'ancienne valeur

useEffect — code qui tourne à un moment précis

// Tourne une seule fois au chargement
useEffect(() => {
  const user = getUser()
  setUser(user)
}, [])  ← tableau vide = une seule fois

// Tourne à chaque fois que 'text' change
useEffect(() => {
  const el = textareaRef.current
  if (!el) return
  el.style.height = 'auto'
  el.style.height = `${el.scrollHeight}px`  // auto-resize
}, [text])  ← rerun quand text change

useRef — garder une référence sans re-render

// Référence vers un élément DOM
const textareaRef = useRef<HTMLTextAreaElement>(null)
<textarea ref={textareaRef} />
// Accès : textareaRef.current.style.height = '40px'

// Garder une valeur sans déclencher de re-render
const filterRef = useRef('pending')
filterRef.current = 'approved'  // pas de re-render

🌐 Appels API — lib/api-client.ts

Toutes les requêtes HTTP passent par 3 fonctions centralisées :

apiFetch — requêtes JSON

// GET simple
const res = await apiFetch('/api/tickets')
const data = await res.json()

// POST avec body
const res = await apiFetch('/api/tickets', {
  method: 'POST',
  body: JSON.stringify({ category: 'Livraison', ... })
})

// PATCH pour modifier
await apiFetch(`/api/tickets/${id}`, { method: 'PATCH', body: JSON.stringify({status:'CLOSED'}) })

apiFetch ajoute automatiquement le header Authorization: Bearer TOKEN et Content-Type: application/json. Si la réponse est 401 (token expiré) → déconnexion automatique.

apiUpload — envoi de fichiers

const fd = new FormData()
fd.append('files', fichier)
await apiUpload(`/api/tickets/${id}/attachments`, fd)
// Pas de Content-Type manuellement — FormData le gère seul

// Lire l'utilisateur connecté

🐍 FastAPI — C'est quoi exactement ?

FastAPI est un framework Python pour créer des APIs. Il reçoit des requêtes HTTP, les traite, et renvoie des réponses JSON.

Structure d'un endpoint (route)

from fastapi import APIRouter, Depends, HTTPException
from ..deps import get_current_user, require_roles

router = APIRouter()

@router.post("/tickets")               ← méthode HTTP + chemin URL
def create_ticket(
    body: TicketCreate,               ← données reçues (validées par Pydantic)
    db: Session = Depends(get_db),    ← connexion DB injectée automatiquement
    current_user = Depends(require_roles(["CLIENT"]))  ← vérifie le rôle
):
    ticket = Ticket(
        client_id = current_user.id,
        category  = body.category,
        ...
    )
    db.add(ticket)
    db.commit()
    db.refresh(ticket)
    return ticket             ← automatiquement converti en JSON

Comment les routes sont déclarées dans main.py

# Dans main.py — chaque router est importé et monté
app.include_router(tickets.router, prefix="/api")
# → POST /api/tickets, GET /api/tickets, etc.

app.include_router(ws.router)
# → WebSocket /ws/ticket/{id} (pas de /api prefix)

✅ Schemas Pydantic — Validation des données

Les schemas sont les "contrats" entre le frontend et le backend. Pydantic valide automatiquement les données reçues et génère les réponses.

class TicketCreate(BaseModel):     ← Schema pour créer un ticket (données REÇUES)
    category:       str
    subcategory:    Optional[str] = None
    ship_name:      Optional[str] = None
    description:    str
    urgency:        str = "MEDIUM"   ← valeur par défaut

    model_config = ConfigDict(
        populate_by_name=True,
        alias_generator=to_camel    ← ship_name → shipName (camelCase pour le frontend)
    )

class TicketOut(BaseModel):         ← Schema pour la réponse (données RENVOYÉES)
    id:         str
    ref:        str
    category:   str
    status:     str
    created_at: datetime

    model_config = ConfigDict(from_attributes=True)  ← lire depuis l'objet SQLAlchemy
ℹ️
camelCase vs snake_case Python utilise snake_case (ship_name), JavaScript utilise camelCase (shipName). L'alias_generator=to_camel fait la conversion automatiquement. C'est pourquoi quand tu vois ticket.shipName côté frontend, c'est ticket.ship_name côté backend.

def require_roles(roles: list[str]):
    def dependency(current_user = Depends(get_current_user)):
        if current_user.role not in roles:
            raise HTTPException(status_code=403, detail="forbidden")
        return current_user
    return dependency

🔄 Migrations Base de données — Comment ajouter une colonne

On n'utilise pas Alembic (outil de migration classique). À la place, des ALTER TABLE sont exécutés automatiquement au démarrage du serveur dans main.py :

# Dans startup() — main.py
with engine.connect() as conn:
    conn.execute(text("""
        ALTER TABLE tickets ADD COLUMN IF NOT EXISTS vessel_data TEXT
    """))
    conn.execute(text("""
        CREATE TABLE IF NOT EXISTS financial_projections (
            id VARCHAR(36) PRIMARY KEY,
            period VARCHAR(7) NOT NULL UNIQUE,
            ...
        )
    """))
    conn.commit()
⚠️
Ajouter une nouvelle colonne — procédure 1. Ajouter la colonne dans le modèle models.py
2. Ajouter le ALTER TABLE ... ADD COLUMN IF NOT EXISTS dans main.py startup()
3. Redémarrer le backend — la colonne est créée automatiquement
Ne jamais faire d'ALTER TABLE directement en prod sans l'ajouter dans startup() !

🔴 Erreurs Frontend courantes — Causes & solutions

TypeError: Cannot read properties of undefined (reading 'firstName')

L'objet existe en DB mais arrive null côté frontend à cause d'un timing.

✅ Solution
Utiliser l'optional chaining : ticket?.client?.firstName ?? '' au lieu de ticket.client.firstName
Input loses focus on every keystroke

Un composant est défini INSIDE le composant parent.

✅ Solution
Déplacer la définition du sous-composant EN DEHORS du export default function. Voir section "Composants React".
hydration mismatch / Text content does not match

Le HTML généré côté serveur est différent du HTML côté client (ex: date affichée différemment).

✅ Solution
Utiliser useEffect pour initialiser les valeurs dépendantes du client (localStorage, Date.now()), pas directement dans le render.
Error: params.locale is undefined

params non traité comme une Promise (Next.js 16).

✅ Solution
useEffect(() => { params.then(p => setLocale(p.locale)) }, [params])
TypeScript Error: Property 'X' does not exist on type 'Y'

Tu utilises un champ qui n'est pas déclaré dans l'interface.

✅ Solution
Ajouter le champ dans l'interface TypeScript. Ex: si form.eta ne compile pas → ajouter eta: string dans interface FormState.
fetch failed / Network Error

Le backend est en panne ou l'URL de l'API est mauvaise.

✅ Solution
1. Vérifier que le backend tourne (http://localhost:8000 en local)
2. Vérifier NEXT_PUBLIC_API_URL dans les variables Vercel
3. Vérifier CORS dans ALLOWED_ORIGINS
Tooltip / popup coupé(e) — seule la première ligne s'affiche

Un élément positionné en absolute est clippé par le overflow: hidden ou overflow-x: auto d'un parent. Ça arrive souvent dans les tableaux avec défilement horizontal.

✅ Solution — utiliser position: fixed + getBoundingClientRect()
Au lieu d'un tooltip CSS (group-hover:block + absolute), utiliser un état React + position: fixed calculé via getBoundingClientRect() pour sortir totalement du contexte overflow : 
// 1. Déclarer l'état
const [tooltip, setTooltip] = useState<{data: any; x: number; y: number} | null>(null)

// 2. Sur l'élément déclencheur
<div
  onMouseEnter={(e) => {
    const r = e.currentTarget.getBoundingClientRect()
    setTooltip({ data: monObjet, x: r.left, y: r.top })
  }}
  onMouseLeave={() => setTooltip(null)}
>

// 3. Tooltip rendu AU NIVEAU DU COMPOSANT RACINE (hors tableau)
{tooltip && (
  <div style={{ position:'fixed', top: tooltip.y - 150, left: tooltip.x - 240, zIndex:9999, pointerEvents:'none' }}
    className="bg-[#1B3A5C] text-white rounded-xl p-3 shadow-2xl w-60"
  >
    {/* contenu du tooltip */}
  </div>
)}
Règle : tout élément avec overflow: hidden/auto/scroll coupe ses enfants en absolute. Seul fixed échappe à ce contexte car il se positionne par rapport à la fenêtre du navigateur.
Dropdown / modal ne s'affiche pas au-dessus d'un tableau

Même cause que le tooltip — le overflow-x: auto du wrapper tableau crée un "stacking context" qui clip tout ce qui déborde.

✅ Solutions possibles
Option A : même pattern position: fixed + getBoundingClientRect() ci-dessus.

Option B : utiliser un z-index très élevé ET changer le wrapper tableau en overflow: visible + gérer le scroll différemment.

Option C : rendre le dropdown en dehors du tableau avec React Portals (ReactDOM.createPortal()) — solution la plus propre mais plus complexe.

Comment un rôle est attribué à la création de compte

Quand quelqu'un crée un compte via le formulaire d'inscription, le rôle est toujours fixé à CLIENT automatiquement par le code — personne ne peut choisir son propre rôle :

🌊 La page d'animation (Splash Screen) — Comment elle est faite

L'écran de chargement au démarrage ("Eolis Connect — Global Logistics Platform" avec les vagues) est défini dans src/app/layout.tsx. C'est du HTML+CSS pur injecté en ligne — pas un composant React — pour qu'il s'affiche immédiatement même avant que React se charge.

Anatomie complète

/* 3 animations CSS définies dans splashCSS */

/* 1. Texte qui monte en fondu (titre + sous-titre) */
@keyframes splashUp {
  from { opacity: 0; transform: translateY(16px); }  ← part de en bas + invisible
  to   { opacity: 1; transform: translateY(0); }      ← arrive en place + visible
}

/* 2. Points qui rebondissent (loader) */
@keyframes sdBounce {
  0%, 80%, 100% { transform: scale(.6); opacity: .4; }  ← petit + transparent
  40%           { transform: scale(1);  opacity: 1;  }  ← grand + visible
}
/* Les 3 points ont un délai différent (0s, .2s, .4s) → effet cascade */

/* 3. Vagues qui bougent horizontalement */
@keyframes swMove {
  0%   { transform: translateX(0); }
  100% { transform: translateX(-50%); }
/* L'astuce : chaque vague fait 200% de largeur (2 copies côte à côte)
   En déplaçant de -50%, quand la 1ère copie sort, la 2ème prend sa place
   → boucle infinie sans saut visible */
}

Comment le splash disparaît — SplashHider.tsx

// SplashHider est un composant React qui tourne côté client
// Il attend que la page soit chargée, puis fait disparaître le splash

export function SplashHider() {
  useEffect(() => {
    const splash = document.getElementById('eolis-splash')
    if (!splash) return

    setTimeout(() => {
      splash.style.opacity = '0'              ← commence le fondu
      splash.style.transition = 'opacity 0.4s ease'
      setTimeout(() => splash.remove(), 400) ← supprime après le fondu
    }, 800)  ← attendre 800ms pour que la page se rende d'abord
  }, [])
  return null  ← ne rend rien visuellement
}
